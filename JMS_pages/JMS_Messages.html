JMS Message

JMS messages are composed of three parts being headers, properties, and body. We will discuss all three parts of the messages in detail.

1. JMS Headers

Headers are used to identify many useful information about the message like, the producer of the message, the destination of the message, when was the message produced and many other information which can be used to keep a track of the message. We make the use of setter methods to set the headers value. The setter methods will be discussed with their respective headers. There are various headers that can be used depending upon the developer’s requirement, below we will discuss a few important headers.

  • JMS MessageID:

    • All JMS messages have a unique ID to identify each message sent by the provider. The message ID is a string value with a unique set of characters. We make the use of setJMSMessageID setter method to set or update the value of JMSMessageID header. Since message ID carries an overhead, the applications not using the message ID, the JMS producer can give a hint to JMS provider to suppress the message id to null by using setDisableMessageID method.

  • JMS CorrelationID:

    • In order to keep track of the message flow throughout the application architecture we use the JMSCorrelationID headers. A typical example would be in the scenario where we need to link a response to a particular request message. setJMSCorrelationID is the setter method to set the correlation id header of the message.

  • JMS ReplyTo:

    • This header field contains the destination value (queue or topic) to which the reply of the message should be sent. setJMSReplyTo setter method to set the value of the destination in this header.

  • JMS Expiration:

    • This header is used to set the time at which the message will expire. JMS does this by setting the time to live field whose value is specified while JMS producer is send thing message using send() method. The setter method setJMSExpiration is used to set the expiration value of the message.

  • JMS Priority:

    • JMS defines 10 priority levels to the messages in the range of 0-9, 0 being the lowest and 9 being the highest priority. This is done to expedite the message delivery before the low priority messages. setJMSPriority setter method is used to set the priority of the messages by JMS producer.

    Note: There are various other JMS header fields that can be used as per the user’s requirements: JMSDestination, JMSDeliveryMode, JMSTimestamp, JMSRedelivered, JMSDeliveryTime etc.


    2. JMS Properties


    These are additional and optional header fields which we can add to the JMS messages. There are basically three categories of properties that can be added to a JMS message:

  • Application Specific properties
  • Standard Properties
  • JMS provider properties

    • All JMS defined properties starts with initials JMSX as their prefix. Below are some JMS defined properties:

    1. JMSXUserID
    2. JMSXAppID
    3. JMSXDeliveryCount
    4. JMSXGroupID
    5. JMSXGroupSeq
    6. JMSXProducerTXID
    7. JMSXConsumerTXID
    8. JMSXRcvTimestamp
    9. JMSXState

    2. JMS Body

    JMS provides five different types of messages that can be sent using JMS providers.

  • Stream Messages

    • These messages contains the stream of Java primitive values.

  • Map Messages

    • These types of messages are composed of name-value pair where name contains the String objects while the values contain the java primitive types.

  • Text Messages

    • These messages body contains the String values.

  • Object Messages

    • These message types contains the serializable Java object.

  • Byte Messages

    • These messages contains the stream of uninterrupted bytes.

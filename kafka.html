This tutorial is designed to provide information about JMS and provides a step-by-step procedure on how to implement JMS in your project.
Prerequisite for this tutorial would be to have basic knowledge on object-oriented paradigms and Java.

1. Introduction

JMS stands for Java Message Service which is a standard to access messages in messaging middleware. JMS provides an API to create, send and read messages on Java servers.

1.1 Messaging

It is a process of exchanging information between two or more applications. Applications can communicate with each other by passing or exchanging information (known as message). Messages are asynchronous requests or responses that will be generated by an application and may be consumed by other applications.

1.2 Messaging Domains

There are two types of messaging model that JMS supports:

  • Point to point (PTP) – In this model the messages are created by an application and pushed into the Queue, the queue holds the messages until consumed by another application.


  • Publish/Subscribe (Pub-Sub) – In this model, the message is created by an application and published to a Topic, the same message is consumed by many other applications whenever required.

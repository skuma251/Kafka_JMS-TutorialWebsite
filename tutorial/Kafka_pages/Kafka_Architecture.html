Kafka Architecture

This topic provides us with the details of the structure of Kafka and its composition.

Elements of Kafka

  • Publisher/Producer – Every Kafka Ecosystem have multiple Publishers and these publishers can send messages or publish data to the topic.
  • Messages – The information that needs to be exchanged between the applications.
  • Kafka-Server – When there is an application or bin when you spin up, you will get a kafka server. In kafka Server we can have multiple topics and for every topic there are partitions
  • Consumer Group – There can be a multiple consumer groups inside the Kafka Ecosystem. Each Consumer can have one or more than one Consumers present inside it. The role of the consumer is the read or consume the message from the topics.

    • Kafka holds key-value messages that arrive from producers. The information obtained may be broken down into many sections under various topics. "Within the individual sections messages are sorted by their position in that section, also there are indexed and saved with a timestamp."(documentation kafka)

    "A Topic is the subject identifier of a category of messages or events."(Sookocheff.,2015)
    A Partition is same as how we have disc partition, we do this partition because it helps event streams to provide high throughput


    "Kafka Replication Factor refers to the multiple copies of data stored across several Kafka brokers. Setting the Kafka Replication Factor allows Kafka to provide high availability of data and prevent data loss if the broker goes down or cannot handle the request."(hevodata.,2020)

    Info About Zookeeper:

    Zookeeper is an application present in the ecosystem. "Zookeeper is a distributed, open-source configuration system and a synchronization service."(edureka.,2019) All the topics data, consumer groups etc are stored in Apache Zookeeper. The word configuration came because when ever there is a change in the configurations of the files it automatically synchronizes the changes throughout the ecosystem. It also says which messages the consumer read, Topic information, Network information of the cluster etc.

    Kafka API

    kafka has four common APIs for sending and receiving messages.(LinkedIn.,2011)

  • Producer API – Giving permission to an application to send the data streams.

  • Consumer API – Giving permission to an application to consume to the topics and access the data streams.

  • Connector API – It processes the producer, consumer APIs which connect the topics to the existing application.

  • Streams API – The result is produced by this API after converting the input streams to output.

    • Benifits of Kafka(confluent)

  • Fault Tolerant - Ability of a system to continue operating without interruption when its one or more components got failed.
  • Scalability - You can scale the kafka system based on the load requirements by adding new brokers and also we can increase the number of consumers for testing whether it is working good for heavy loads.
  • Durability - "Kafka messages uses logs" and are generally stored on the secondary storages and can be accessed quickly, hence durable.
  • Reliability - "Kafka is well distributed, clusters all the information like messages, bytes etc and is risk tolerant"
  • Performance - "Kafka has high efficiency on both sides publishing and subscribing messaging" i.e., it can control large amount of data and messages easily.

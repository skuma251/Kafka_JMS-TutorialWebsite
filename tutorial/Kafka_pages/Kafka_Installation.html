Kafka Installation Steps using Docker

This topic provides us with the details of the installation steps of Kafka.

We have to have docker installed in the machine. For installation go through this documentation and install docker for your respective operating system docker_installation.

After installing docker, we have to create a docker_config.yml file where the kafka and zookeeper configuration, ports and relevant settings will be written. So that virtual environment is created.
In that environment we can add Kafka Container and Zookeeper Container. Advantage for this .yml file is we can keep this file in any Operating system and install the Apache Kafka and Zookeeper with those respective settings.

Explanation for the above configurations

We have Kafka container and Zookeeper container because they both make a connection in between them and shares some of their properties so that Kafka broker works properly. The use of Zookeeper over here is because Kafka needs Zookeeper for different type of management like selection, storing data, count of the consumer work etc. External world can connect to this Kafka and Zookeeper container using the boundaries such as ports

In Services we have Zookeeper and Kafka to be written and the image is from the dockerhub. Then after writting the yml file type 'docker-compose -f docker-compose.yml up -d' in the terminal below in your IDE so that apache kafka and zookeeper will be initiating

After that it's done. Sit back and relax your kafka and zookeper and running in the background you can check the status of them in the docker app. Else you can type a command 'docker ps' in your terminal it will list all the running servers.

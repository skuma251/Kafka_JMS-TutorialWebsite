Point to Point Messaging

The main goal of Point to point messaging is when you want only one consumer to receive the message at a time.

Process:

Messages are sent via a queue. Any number of producers can send a message onto the queue. But, a message can be accessed by one consumer only.

  • When is it used?

    • PTP is specifically used when we want only one consumer to access the message at a time.

  • Queue ordering:

    • Messages are ordered on the basis of when they enter the queue.

  • Advantages of PTP:

    • 1. Messages do not get lost even in the case when there are no receivers for that particular message.

    2. A queue browser can be used if the client is a Java client. This way, contents can be continuously checked on the queue.

    3. A message selector can be used to get messages that are not present at the head of the queue.

1. Introduction

Apache Kafka is a distributed event driven and stream processing platform. Kafka is an open-source platform developed by Apache Software Foundation written in Java and Scala. "Streaming data means the data that is frequently provided by hundred of sources, whoch essentially sends the data records in at same time."(AWS).

2. History

"Kafka was originally developed by LinkedIn and was subsequently open sourced in early 2011. Jay Kreps, Neha Narkhede and Jun Rao helped co-create Kafka. Jay Kreps chose to name the software after the author Kafka because it is “a system optimized for writing”, and he liked Kafka’s work"(LinkedIn., 2011).

3. Messaging Domains

Messaging system is a median where the data bits or the messages are sent to and fro between the sender and reciever.
There are two types of messaging model that Kafka supports:

  • Point to point (PTP) – In this model the messages are "created by an application and pushed into the Queue, the queue holds the messages until consumed by another application."(tutorialspoint, 2022).


  • Publish/Subscribe (Pub-Sub) – In this model, the message is "created by a prodcuer and published to a Topic, the same message is consumed by many other applications whenever required using a single destination."(tutorialspoint, 2022).

    • 4. Kafka

    "Apache Kafka is a distributed publish-subscribe message system and a powerful queue that can control high amount of information and allows us to pass messages to and fro."(tutorialspoint) Kafka messages are generally stored on the secondary storages and used within the cluster to eliminate data redundancy and loss. In production environment Kafka is referreed as Kafka Clusters. A cluster is the one which consists of more than one Kafka server. And each Kafka server is referred to as a broker.
